# Methodology

## Two-step framework

Following your poster, the intended use is:

1. **Generate plausible patients** using your QSP model and biological constraints.
2. **Select a virtual cohort** with optimization so clinical endpoint summaries are matched.

`VPopMIP.jl` currently focuses on step 2.

## Optimization model

For each candidate patient `i`, define a binary selection variable:

- `x_i = 1` if candidate `i` is included in the selected VPop
- `x_i = 0` otherwise

Subject to:

- **Cohort size**: `sum(x_i) = N` where `N` is requested VPop size.
- **Preselected candidates** (optional): enforce `x_i = 1` for marked patients.

Objective:

- Minimize total mismatch across all endpoint/metric targets.
- Each target contributes a mismatch expression generated by `DigiPopData.add_mismatch_expression!`.

## Endpoint types (current support)

Through `DigiPopData` metric definitions:

- Mean
- Mean + standard deviation
- Quantiles
- Survival curves

These align with the endpoint classes in your poster (e.g., baseline summary, best response quantiles, PFS/survival).

## Practical considerations

- The MIP may become expensive for large plausible populations.
- Feasibility and fit quality depend strongly on diversity of candidate patients.
- If no optimal solution is found under current settings, consider:
  - relaxing endpoint requirements,
  - increasing plausible population diversity,
  - changing solver settings/time limits.

## Solver notes

By default, `SCIP.Optimizer` is used. You can provide a custom optimizer and time/gap settings in `select_cohort(...; kwargs...)`.
